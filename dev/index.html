GpuTestEnv

GpuTestEnv helps downstream packages create a GPU-aware test environment without hard-coding every GPU backend into their main test dependencies.

The core workflow:

  1. Copy or create a temporary test environment from test/Project.toml.
  2. Develop the parent package into that environment.
  3. Predict which GPU backends are worth attempting on the current host.
  4. Install those backends.
  5. Run each backend's runtime-availability check and keep only the usable ones.

Both modern test environments based on test/Project.toml and legacy test-only dependencies defined with [extras] and [targets] in the root project are supported.

Backend prediction prefers direct host hints such as Linux device nodes and PCI vendor IDs, Windows video-controller names, and macOS display metadata. Command-line tools like nvidia-smi, rocminfo, sycl-ls, and clinfo remain fallback probes.

Test environment example

# test/runtests.jl
using GpuTestEnv

const ROOT = normpath(joinpath(@__DIR__, ".."))
GpuTestEnv.activate(; path = ROOT, persist = true)

On Julia 1.10, a separate bootstrap environment is only needed if the downstream test environment cannot be resolved before GpuTestEnv.activate(...) gets a chance to remove newer-only dependencies.

Benchmarking environment example

Run the script as julia --project=benchmark benchmark/gpu_benchmark.jl.

# benchmark/gpu_benchmark.jl
using GpuTestEnv

const ROOT = normpath(joinpath(@__DIR__, ".."))
GpuTestEnv.activate("MyPackage"; path = ROOT, persist = false, include_jlarrays = false)

using MyPackage, BenchmarkTools

# Use the first available GPU backend
backend = first(gpu_backends(; include_jlarrays = false))
x_gpu = backend(randn(Float32, 1024))

No need to add CUDA or other GPU packages to the benchmark Project.toml; GpuTestEnv detects and installs them during activate. While GpuTestEnv is not yet registered, keep a [sources] path entry for it in the benchmark project.

Working with backends after activation

julia> using GpuTestEnv

julia> fake_loader = backend -> backend == :JLArrays ? (Main, Int) : nothing;

julia> GpuTestEnv.backend_modules_and_array_types([:JLArrays, :CUDA]; importer = fake_loader)
1-element Vector{Tuple{Module, DataType}}:
 (Main, Int64)

Query the available backends after activate has run:

backends = gpu_backends()                          # JLArrays + all native backends that resolve here
backends_fftw = gpu_backends(; supports_fftw = true)  # only FFTW-compatible
all_tested = gpu_backends(
    ;
    include_jlarrays = false,
    backends_to_test = collect(supported_backends(; include_jlarrays = false)),
)

for backend in backends
    x = gpu_randn(backend, Float32, 128, 128)
    @show backend.name, typeof(x)
end

backends_to_test only narrows the backend probe/query set. Use it when you want to skip known-irrelevant backends or limit the default all-native query set to a smaller subset such as CUDA and AMDGPU.

synctestenv versus activate

Use sync_test_env when you want a SyncResult describing the GPU environment but you do not want to leave that environment active. Use activate when the synchronized GPU environment should stay active for the rest of the current Julia session.

JLArrays opt-out

JLArrays is enabled by default. Skip it when testing native GPU paths only:

GpuTestEnv.activate(; path = ROOT, include_jlarrays = false)
